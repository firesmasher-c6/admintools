How It Works

Firebase Realtime Database

Your knowledge is stored in Firebase with this structure:
opticore-client-portal-default-rtdb
/limitx1/knowledge.json
/limitx2/knowledge.json

Knowledge Structure

{
  "knowledge_batch_1": "Core facts and definitions",
  "knowledge_batch_2": "Procedures and workflows",
  "knowledge_batch_3": "Background context",
  "knowledge_batch_4": "Examples and cases",
  "important_knowledge": "Critical rules",
  "last_updated": "2025-01-15T..."
}

How Workers Use Knowledge

  1. User asks question → Sent to Cloudflare Worker
  2. Worker fetches knowledge → From Firebase database
  3. Worker adds context → Includes knowledge in prompt
  4. Gemini API responds → Using the knowledge
  5. Answer sent back → To your chat interface
